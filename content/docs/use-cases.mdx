# Use Cases

The S0 IoT Gateway is designed for diverse IoT applications across smart metering, industrial automation, agriculture, and edge computing. Here are detailed use cases demonstrating the platform's versatility.

## Smart Metering

### Residential Utility Monitoring

**Challenge**: A utility provider needs to remotely monitor electricity, gas, and water usage across hundreds of homes. Some homes have legacy wired M-Bus meters, while others use modern wireless M-Bus models.

**Solution**:

- Deploy S0 gateway in each apartment block
- DIN-rail mount inside electrical cabinets using Base Board
- Use Wireless M-Bus (RC-S2LP) to read meters without rewiring
- Use Wired M-Bus via Base Board for legacy devices
- Backhaul readings to utility's cloud via NB-IoT (SIM7080G)
- Periodic uploads to Magistrala using MQTT
- Retry logic ensures resilience in poor cellular areas
- Power from 24V control panel supply with battery backup

**Benefits**:

- No truck rolls for meter reading
- Real-time consumption monitoring
- Leak detection and anomaly alerts
- Support for mixed meter infrastructure
- Reliable operation with battery backup

**Technical Details**:

```yaml
Hardware:
  - S0 Gateway with NB-IoT module
  - Base Board with M-Bus transceiver
  - External antenna for basement installation

Connectivity:
  - Wireless M-Bus: 868 MHz, Mode S1
  - Wired M-Bus: 300-9600 baud
  - NB-IoT: 15-minute update interval

Data Flow: 1. Collect meter readings every 15 minutes
  2. Log to SD card for redundancy
  3. Aggregate and compress data
  4. Upload via NB-IoT to Magistrala
  5. Cloud processes and bills customers
```

## Industrial IoT

### Predictive Maintenance Platform

**Challenge**: A manufacturing facility needs to monitor industrial machinery for early signs of failure to minimize downtime and maintenance costs.

**Solution**:

- Deploy S0 gateways on critical equipment
- Collect high-frequency sensor data:
  - Vibration sensors via I2C
  - Temperature sensors via SPI
  - Pressure sensors via analog inputs
- Local ML inference for real-time decisions
- Send only alerts to central system
- Historical data logging to SD card

**Benefits**:

- Catch failures before they occur
- Reduced network bandwidth usage
- Real-time response to critical conditions
- Reduced downtime and maintenance costs
- Local operation during network outages

**Technical Details**:

```yaml
Edge Processing:
  - Wasm workload: 50KB FFT analysis
  - Update rate: 10 Hz sensor sampling
  - ML inference: <50ms latency
  - Alert generation: Local threshold detection

Connectivity:
  - Primary: Ethernet via Base Board
  - Backup: WiFi
  - Emergency: NB-IoT SMS alerts

Data Management:
  - Local buffering: 7 days on SD card
  - Upload: Alerts immediately, trends hourly
  - Retention: 30 days cloud, 7 days local
```

## Agriculture & Environment

### Precision Agriculture Gateway

**Challenge**: A farm operator wants to monitor soil moisture, weather conditions, and water flow across large fields far from any infrastructure.

**Solution**:

- Deploy S0 in weatherproof enclosures
- Solar panel with battery backup
- Sensors via I2C/SPI/UART interfaces:
  - Soil moisture sensors
  - Weather station
  - Water flow meters
- GNSS (via SIM7080G) for geo-tagging
- NB-IoT for data backhaul
- SD card for offline storage
- Local processing for irrigation control

**Benefits**:

- Optimize water usage
- Remote field monitoring
- Automated irrigation control
- Historical data analysis
- Works in areas without infrastructure

**Technical Details**:

```yaml
Deployment:
  - IP67 enclosure
  - 20W solar panel
  - 12V 20Ah battery
  - External LTE antenna on mast

Sensors:
  - 6x soil moisture (I2C)
  - 1x weather station (UART)
  - 1x flow meter (pulse counter)
  - 1x GNSS module (internal)

Operation:
  - Sample interval: 15 minutes
  - Upload interval: 1 hour
  - Emergency alerts: Real-time via SMS
  - Battery life: 7 days without sun
```

## Edge Computing

### Edge AI Inference Platform

**Challenge**: Run AI inference at the edge for real-time decision making in security, quality control, or process monitoring applications.

**Solution**:

- S0 + BeagleV-Fire HAT configuration
- Camera or sensor array connected to BeagleV-Fire
- S0 provides connectivity and power management
- TensorFlow Lite inference on Linux
- Results sent via optimal connectivity

**Benefits**:

- Real-time inference (less than 100ms)
- Privacy-preserving (data stays local)
- Works offline
- Reduced cloud costs
- Low bandwidth usage

**Technical Details**:

```yaml
Architecture:
  - BeagleV-Fire: ML inference (TFLite)
  - S0: Connectivity + data collection

Performance:
  - Inference: 50ms typical
  - Preprocessing: 20ms (Wasm)
  - Total latency: less than 100ms edge-to-decision

Models:
  - Object detection: MobileNet SSD
  - Classification: ResNet-18 (quantized)
  - Anomaly detection: Autoencoder
```

### Protocol Translation Gateway

**Challenge**: Bridge legacy industrial protocols (Modbus, BACnet) with modern IoT platforms (MQTT, HTTP).

**Solution**:

- S0 collects data from legacy devices

- Unified MQTT output to Magistrala
- Bidirectional control

**Benefits**:

- Integrate legacy systems
- Modernize without replacement
- Flexible protocol support
- Centralized management

## Deployment Configurations

### Standalone Gateway

- **Use**: Simple data collection
- **Power**: Battery or USB
- **Connectivity**: WiFi, BLE, NB-IoT
- **Cost**: Lowest
- **Complexity**: Minimal

### With Base Board

- **Use**: Industrial installation
- **Power**: 24V DC + battery backup
- **Connectivity**: + Ethernet, M-Bus
- **Features**: + SD card logging
- **Mounting**: DIN-rail

### With BeagleV-Fire

- **Use**: Edge computing, AI
- **Power**: 5V/3A
- **Processing**: Full Linux
- **Features**: Advanced applications
- **Development**: Standard tools
